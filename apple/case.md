# Дополнительное задание
## Детекция логотипа Apple

### Цель

Создать REST API сервис, который сможет обнаруживать логотип Apple на загружаемых изображениях и возвращать координаты найденных логотипов.

Необходимо разработать систему автоматического поиска и детекции логотипа Apple на изображениях. Логотип представляем собой надкусанное яблоко (при этом цвет может быть произвольный). Для этого потребуется некоторым образом собрать данные, разметить и собрать на их основе свое решение. Так же необходимо составить свой валидационный набор данных и реализовать подсчет качества своего решения на нем.

### Данные

В этом задании данные нужно подготовить самостоятельно, это включает в себя и скреппинг данных из Интернета, их разметку и прочие необходимые шаги.

Разрешается использовать открытые датасеты, если таковые есть.

### Ограничения

* Время обработки: не более `10` секунд на изображение
* Железо: Возможность запуска на видеокарте с 16GB видеопамяти (уровня google collab T4)
* Поддерживаемые форматы: `JPEG, PNG, BMP, WEBP`

### Ожидаемый формат решения

По результатам работы необходимо предоставить публичный GitHub репозиторий с полным исходным кодом решения. Решение должно быть упаковано в докер контейнер, который запускается одной командой и поднимает rest api сервис на порту `8000`. В корне репозитория обязательно должен находиться файл `README.md` с подробными инструкциями по сборке и запуску контейнера.

Дополнительно в ридми необходимо описать подход к решению задачи с достаточной степенью подробности. **Веса обученной модели, валидационные данные необходимо выложить на любой открытый файлообменник и приложить ссылки на их скачивание в ридми.**

Так же необходимо реализовать скрипт валидации качества решения и подсчета метрики (F1-score, `iou=0.5`) на отобранных и размеченных примерах.

### Подсказки

* Используйте **zero-shot** подходы для подготовки разметки данных.
* Обратите внимание на предобработку изображений и обработку negative примеров.
* Тестируйте решение на различных типах изображений, уделите время проверке качества разметки вашей валидационной выборки.
* Документируйте свои эксперименты и выбор подхода в процессе работы.
* Обратите внимание на качество оформления ридми, пишите свои шаги по решению максимально подробно.

Удачи!
